# NotebookLite Documentation

## Introduction

NotebookLite is a local-first, privacy-respecting intelligence platform designed for reading, analyzing, and revising PDF documents. The application allows users to upload documents, perform Retrieval-Augmented Generation queries, view source PDFs side-by-side with assistant answers, and automatically generate key concepts, study guides, and interactive quizzes.

The system is built to run locally and integrates with Google Gemini or OpenRouter APIs for generation. It features a complete offline fallback using local mathematical vector representations to operate without internet connections or API keys.

## Architecture and Directory Structure

The workspace is split into two primary components: the FastAPI backend and the React frontend.

1. backend: Contains the FastAPI application server, document processing pipelines, and local vector storage logic.
   * app.py: Entry point for the FastAPI application. Sets up routes, CORS, static file serving for PDF viewports, and coordinates retrieval requests.
   * pdf_processor.py: Extracts raw text page-by-page from uploaded PDF documents using PyMuPDF.
   * rag_engine.py: Houses the chunking parser, local hashing vectorizer, Gemini embedding integration, NumPy cosine similarity logic, and the multi-provider LLM response router.
   * uploads: Directory where PDF files are stored and served statically to the viewer.
   * vector_store: Directory where processed document chunk JSON data and numerical embeddings are saved.

2. frontend: Vite-powered React single-page application.
   * src/App.jsx: Root component managing workspace state, split-pane layout, local storage synchronization, API requests, and coordinate anchors.
   * src/index.css: Tailored Tailwind CSS stylesheet implementing the Catppuccin Mocha soft dark theme with rounded layout elements optimized for low GPU rendering overhead.
   * src/components/Sidebar.jsx: Manages source file uploads, selection states, indexing actions, and storage metrics.
   * src/components/Navbar.jsx: Governs workspace view switching between Chat and Study modes.
   * src/components/ChatWindow.jsx: Feeds conversational history, handles text inputs, and displays processing states.
   * src/components/ChatBubble.jsx: Formats message history and parses citation markers into interactive click targets.
   * src/components/StudyGuide.jsx: Displays summary cards, key concepts, interactive quiz questions, and 3D flipping flashcards.
   * src/components/SettingsModal.jsx: Allows configuring the API provider, entering API keys, and selecting active LLM models.

## Core Capabilities

1. High-Performance Dual-Pane PDF Viewer
   Clicking on citation markers in the assistant bubble or clicking the view icon in the sidebar opens the PDF statically in a split viewport. The layout leverages native browser rendering. The app automatically scrolls the PDF to the exact page cited by using standard page hash navigation.

2. Multi-Provider LLM Integration
   The platform supports Google Gemini and OpenRouter APIs. Users can choose their preferred provider in the settings. This allows routing generation tasks to alternative models (such as DeepSeek or Llama) if Gemini experiences network issues.

3. Offline Local Embedding Fallback
   If no API key is provided, the backend falls back to a deterministic local hashing vectorizer. It generates a 1024-dimension numerical representation of document text using a Python implementation of the DJB2 string hashing algorithm. The similarity search is performed locally via NumPy dot product operations.

4. Low-Spec CPU Optimization
   The interface uses solid background layers and rounded container shapes. It avoids heavy pixel shaders, animations, and backdrop blurs. This keeps CPU and GPU utilization low on integrated mobile graphics processors.

5. Visual Citation Grounding
   All answers generated by the model are mapped back to their original sources. Hovering over a citation displays a tooltip showing the exact text snippet retrieved from the PDF, while clicking the citation scrolls the PDF viewer directly to the source page.

## Technical Setup

1. Requirements
   * Python 3.9 or higher
   * Node.js 18 or higher

2. Backend Installation and Execution
   * Navigate to the backend directory:
     cd backend
   * Create a virtual environment:
     python3 -m venv venv
   * Activate the virtual environment:
     source venv/bin/activate
   * Install the dependencies:
     pip install fastapi uvicorn pymupdf numpy requests pydantic
   * Start the FastAPI server:
     uvicorn app:app --reload

3. Frontend Installation and Execution
   * Navigate to the frontend directory:
     cd frontend
   * Install npm dependencies:
     npm install
   * Start the Vite development server:
     npm run dev
   * Open the application in your browser by visiting http://localhost:5173
